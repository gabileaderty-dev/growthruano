# GLOBAL INSTRUCTIONS FOR ALL LANGUAGES

## Primary goals:
- Always generate SIMPLE, CLEAR, and MINIMAL code.
- Prefer the most straightforward solution that works.
- Avoid overengineering, premature abstractions, and speculative features.

## IMPORTANT RESTRICTIONS:
- DO NOT create additional files beyond what's clearly required to solve the problem.
- DO NOT create standalone documentation (README, docs) or large comment blocks unless I explicitly ask.
- DO NOT add packages, dependencies, or libraries unless I explicitly ask.
- DO NOT generate or modify configuration files (tsconfig, .env, etc.) unless I explicitly ask.
- DO NOT scaffold project structures or boilerplate unless I explicitly ask.
- DO NOT extract abstractions, utilities, or helpers unless code duplication is significant (>3 instances) OR I explicitly ask.

## YOU MUST ASK ME FIRST before:
- Creating new files (including test files, type definitions, or modules)
- Adding new external dependencies
- Modifying project structure or configuration
- Introducing new patterns, layers, or architectural changes

## Code generation rules:
- Write only the code strictly necessary to solve the requested problem.
- Prefer modifying existing files over creating new ones.
- Prefer functions over classes unless a class is clearly required.
- Keep inline comments minimal; use them only for non-obvious logic.
- Use explicit types and avoid any (in TypeScript/typed languages).
- Keep type annotations minimal but correct.

## Interaction rules:
- If the request is ambiguous, ask a clarifying question instead of assuming.
- If something optional could be added, ask before adding it.
- Be concise; do not provide lengthy explanations unless I explicitly ask.
